Each subdirectory under `stacks/` is an independent Terraform module with its own state file; isolation is achieved through unique state prefixes (`stacks/<number>-<name>`) and distinct provider session names. All stacks share a uniform provider contract: they assume the same spoke role via `var.spoke_role_arn`, target the same region, and persist state to an OSS bucket (`tfstate-<CICD_ACCOUNT_ID>-cn-hangzhou`) locked by a Tablestore instance (`tfstate-lock`). The numeric prefixes encode deployment order — identity (10), logging (11), guardrails (12/13), networking (20/21), then security workloads (30) — so CI/CD can apply them sequentially without cross-stack data dependencies. Cross-account access is uniformly handled through `assume_role` blocks rather than per-resource delegation.
