This is a leaf Terraform module under the LZA stacks hierarchy (`stacks/10-identity-cloudsso`) that currently acts as a stub awaiting implementation of CloudSSO directories, SCIM, and access configurations. Internal layout follows the standard single-file-per-concern pattern: `main.tf` documents the intended future composition via `module "cloudsso" { source = "../../modules/lza/components/identity/cloudsso" }`; `providers.tf` configures the `alicloud` provider with cross-account assumption through `assume_role.role_arn = var.spoke_role_arn` using session name `tf-identity-cloudsso` and 3600s expiration; `versions.tf` pins `terraform >= 1.5`, requires `aliyun/alicloud >= 1.262.1`, and declares an `oss` backend scoped to `prefix = "stacks/10-identity-cloudsso"` with Tablestore-based distributed locking at `tablestore_table = "tflock"`; `variables.tf` exposes only `region` (default `cn-hangzhou`) and the required `spoke_role_arn`; `outputs.tf` is reserved for future resource IDs (directory ID, access configuration IDs). Dependency direction is outward-only: this stack depends on the reusable `modules/lza/components/identity/cloudsso` component once implemented, and on the shared OSS/Tablestore state infrastructure provisioned by earlier bootstrap stacks.
