Two parallel leaf stacks mirror each other under `stacks/12-guardrails-preventive/` and `stacks/13-guardrails-detective/`, each following the standard five-file layout (`main.tf`, `providers.tf`, `variables.tf`, `outputs.tf`, `versions.tf`). Both are intentionally empty stubs: `main.tf` contains only a comment pointing at the production module sources `../../modules/lza/components/guardrails/preventive` and `.../detective`, with a TODO marker for future implementation. Each stack declares its own `alicloud` provider in `providers.tf` configured with `assume_role` against `var.spoke_role_arn`, scoped to `var.region` (default `cn-hangzhou`) and a unique `session_name` (`tf-guardrails-preventive` / `tf-guardrails-detective`) with a fixed 3600s expiration, so the two stacks can run concurrently against different accounts without session collision. Inputs are identical across both stacks — `region` and `spoke_role_arn` — keeping the preventive/detective split purely logical rather than infrastructural.
