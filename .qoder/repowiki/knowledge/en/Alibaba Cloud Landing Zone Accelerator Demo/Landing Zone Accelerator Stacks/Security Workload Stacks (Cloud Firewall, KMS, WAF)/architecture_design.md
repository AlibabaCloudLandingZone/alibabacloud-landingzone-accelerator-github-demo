Three independent Terraform stacks under `stacks/30-security-*`, each a leaf module with the standard layout (`main.tf`, `variables.tf`, `providers.tf`, `outputs.tf`, `versions.tf`). Each stack declares an `alicloud` provider configured to assume a spoke role via `assume_role.role_arn = var.spoke_role_arn`, enforcing cross-account spoke-side execution. The `main.tf` files are intentionally empty scaffolds containing commented-out `module "..." { source = "../../modules/lza/components/security/<component>" }` blocks pointing at reusable LZA components (`cloud-firewall`, `kms`, `wafv3`) — production resources will be sourced from those modules rather than defined inline. State is remote-backed per-stack via an OSS backend whose bucket name uses a `<CICD_ACCOUNT_ID>` placeholder and whose `prefix` isolates state by stack path; TableStore provides distributed locking. Inputs are uniform across stacks: `region` (default `cn-hangzhou`) and required `spoke_role_arn`; outputs are reserved as comments awaiting resource IDs once components are wired in.
