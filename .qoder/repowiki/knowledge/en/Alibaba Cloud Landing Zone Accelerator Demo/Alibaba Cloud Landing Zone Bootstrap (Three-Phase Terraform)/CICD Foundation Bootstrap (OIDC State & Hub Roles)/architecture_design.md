Single-module Terraform configuration (`main.tf`) that provisions four layers inside the CICD account: (1) OSS bucket with KMS encryption/versioning plus Tablestore instance/table for remote state and locking; (2) an `alicloud_ims_oidc_provider` pointing at `token.actions.githubusercontent.com`; (3) two RAM roles — `GitHubActionsPlanRole` scoped to `pull_request` events and `GitHubActionsApplyRole` scoped to `environment:production`, both trusting the OIDC provider via `oidc:sub` conditions; (4) a shared `HubStateAccess` policy granting OSS read/write, OTS access, and cross-account `sts:AssumeRole` on spoke `SpokePlanRole`/`SpokeApplyRole`. Three `alicloud` providers are declared: default (direct CICD creds), `mgmt` alias (phase-0 operator creds), and `cicd` alias which chains into the CICD account through `ResourceDirectoryAccountAccessRole`. All resources target `provider = alicloud.cicd` so they land in the member account. `backend.tf.example` documents the deferred migration from local to OSS backend via `terraform init -migrate-state` after obtaining STS credentials for the CICD account.
