Single Terraform module (`bootstrap/00-org-structure`) that provisions the top-level organizational graph consumed by all downstream stacks.
- Entry point `main.tf` runs a three-stage flow: enable `alicloud_resource_manager_resource_directory` (idempotent one-way switch), query its root via `data.alicloud_resource_manager_resource_directories`, then create sibling folders `Core`, `Workloads`, `Sandbox` under that root, and finally instantiate five `Trusteeship`-billing member accounts (`devops`, `log-archive`, `security`, `network`, `shared-services`) inside `Core` using a `for_each` over a local map.
- Interface boundary: `variables.tf` exposes only `region` (default `cn-hangzhou`); `outputs.tf` returns `root_folder_id`, `folder_ids` (map), and `account_ids` (map) for downstream modules to consume.
- Dependency direction: `providers.tf` pins the `alicloud` provider to `var.region`; `versions.tf` constrains `terraform >= 1.5` and `aliyun/alicloud >= 1.262.1`, with an explicit comment that the local backend is temporary and will be migrated to OSS in phase 2.
- No child sub-modules — this leaf owns the entire resource graph and is intended to be applied first before any workload or policy stack.
