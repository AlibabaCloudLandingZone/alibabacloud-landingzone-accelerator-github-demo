The module is a flat collection of sibling Terraform workspaces (`00-org-structure`, `01-cicd-foundation`, `02-spoke-bootstrap`) whose numeric prefixes enforce a strict dependency graph: phase 0 creates the management account's Resource Directory and five core member accounts; phase 1 stands up OSS + OTS state backends and the GitHub Actions OIDC provider with plan/apply hub roles inside the CICD account; phase 2 deploys per-account `SpokePlanRole`/`SpokeApplyRole` into every member account. Cross-child wiring relies on two patterns rather than shared code:
- Contract-bound variables/outputs: phase 0 emits `cicd_account_id`, `spokes`, folder IDs etc., which phases 1–2 consume as inputs.
- Provider alias chaining: phases 1 and 2 declare a default `alicloud` provider plus one aliased provider per target account, each configured with `assume_role` on `ResourceDirectoryAccountAccessRole` to hop from management → target account; phase 2 instantiates the reusable `modules/spoke-roles` module once per alias.
Each child owns its own `.terraform`, `providers.tf`, and isolated local backend, designed to be migrated later to the centralized OSS/OTS state provisioned in phase 1.
