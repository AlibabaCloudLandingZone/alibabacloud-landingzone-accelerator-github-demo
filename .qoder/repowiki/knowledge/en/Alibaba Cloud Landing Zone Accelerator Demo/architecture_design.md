The root is the orchestrator surface for two child modules — `bootstrap/` (phases 0–2: org structure, CI/CD foundation, spoke roles) and `infrastructure_stacks/` (numbered workload stacks). Cross-child wiring happens exclusively through `.github/workflows/`: each bootstrap phase has its own workflow (`bootstrap-00-org-structure.yml`, `bootstrap-01-cicd-foundation.yml`, `bootstrap-02-spoke.yml`) and all of them delegate to the single reusable workflow `terraform-reusable.yml`, which encapsulates the OIDC → Hub Role → Spoke Role STS chain. The `stacks.yml` workflow drives the `infrastructure_stacks` children in parallel via a matrix over `SPOKE_ACCOUNT_IDS_JSON`. Bootstrap outputs (OIDC provider ARN, OSS state bucket, Tablestore lock table, hub role ARNs) are consumed as GitHub repository variables by stack pipelines, enforcing a strict dependency order where no stack can run until Phase 2 provisions the shared backend and trust boundaries.
