Bootstrap must be run manually in three phases before the pipeline takes over: `cd bootstrap/00-org-structure && terraform init && terraform apply`, then repeat for `01-cicd-foundation` and `02-spoke-bootstrap`. After Phase 2 provisions the OSS backend, migrate local state with `terraform init -migrate-state` using the `backend.tf.example` template. Production applies require the `production` GitHub environment to be configured with required reviewers; repository variables `HUB_ACCOUNT_ID`, `GHA_PLAN_ROLE_ARN`, `GHA_APPLY_ROLE_ARN`, `OIDC_PROVIDER_ARN`, and `SPOKE_ACCOUNT_IDS_JSON` must be set.
