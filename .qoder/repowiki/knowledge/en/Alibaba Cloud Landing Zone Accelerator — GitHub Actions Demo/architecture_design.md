The repository is organized into three concentric layers:
- `bootstrap/00-org-structure`, `01-cicd-foundation`, `02-spoke-bootstrap` — sequential, single-account Terraform workspaces that create the Resource Directory (folders + member accounts), then the CICD foundation (OSS state bucket encrypted with KMS, Tablestore lock table, GitHub OIDC provider, hub Plan/Apply roles in the CICD account), and finally spoke roles in each member account. Each phase has its own `.terraform.lock.hcl` and `backend.tf.example` for post-bootstrap OSS backend migration.
- `stacks/<number>-<name>/` — per-workspace Terraform stacks targeting individual spoke accounts; each stack declares `providers.tf`/`variables.tf`/`main.tf` and is driven by the matrix in `.github/workflows/stacks.yml`, which maps a stack to an `account_key` resolved from `SPOKE_ACCOUNT_IDS_JSON`. The `20-network-cen` stack demonstrates direct resource definitions while commenting out the vendored module path as the production pattern.
- `modules/lza/` — vendored copy of the upstream Alibaba Cloud Landing Zone Accelerator, split into reusable `components/*` (account-factory, guardrails, identity/cloudsso, log-archive, network, resource-structure, security) and leaf `modules/*` (vpc, oss-bucket, sls-project, kms-instance, etc.). Each component ships `README.md`/`README-CN.md`, `main.tf`, `outputs.tf`, `variables.tf`, `versions.tf`.
- `.github/workflows/` — CI orchestration: `terraform-reusable.yml` is a `workflow_call` entry point that performs checkout → OIDC credential setup (`aliyun/configure-aliyun-credentials-action`) → `setup-terraform@v3` → init/plan/apply, posting plan diffs as PR comments; the four bootstrap workflows call it per directory, while `stacks.yml` runs a parallel matrix over all stacks, gating apply on the `production` GitHub environment.

Dependency direction is strictly one-way: workflows invoke bootstrap phases in order, then drive stacks; stacks consume only `modules/lza` components — nothing inside `modules/lza` references bootstrap or stacks. State lives in OSS with Tablestore locking, and the credential chain is GitHub OIDC token → Hub Role (CICD) → Spoke Role (member account) → Alicloud provider.
